Here’s the clear, practical way to check your usage limits — because ChatGPT and Codex actually track usage in different places, and this confuses almost everyone.

✅ 1. Check ChatGPT (subscription) usage limits

This is your Plus / Pro chat usage — the limits that affect how many messages or heavy requests you can send.

Where to check

Inside ChatGPT:

  1. Click your profile icon (bottom-left).
  2. Open Settings
  3. Go to My Plan (or Subscription)

You’ll see:

  • Your current plan (Plus / Pro / etc.)
  • Upgrade options
  • Sometimes notices when you’re approaching limits

👉 Important reality:

  • ChatGPT does NOT show a live counter of messages remaining.
  • Limits are dynamic and based on load + model usage.
  • You usually only see limits when you hit or approach them.

This lack of a visible counter is expected behavior — usage isn’t exposed as an exact remaining quota in the UI.

✅ 2. Check Codex usage limits (THIS is separate)

Codex has its own quota system tied to coding workload.

Option A — Web usage page (easiest)

Open:

👉 https://chatgpt.com/codex/settings/usage

This shows:

  • Usage consumed
  • Remaining allowance
  • Credit usage (if applicable)

Recent updates added clearer “limits remaining” displays in Codex dashboards. citeturn0reddit24

Option B — Inside Codex CLI / IDE

If you’re using Codex locally:

/status

This shows:

  • % of quota used
  • 5-hour window usage
  • Weekly usage window
  • Active model + limits

Many developers miss this — it’s the primary built-in tracker.

How Codex limits actually work (important)

Your allowance depends on plan and task size:

  • Limits are shared across local + cloud tasks
  • Larger repos or long sessions consume more quota
  • Simple scripts use only a fraction

For example (Plus plan averages):

  • ~45–225 local messages OR
  • ~10–60 cloud tasks per 5-hour window

(with a shared weekly cap) citeturn0search2

So usage feels inconsistent because context size affects consumption.

✅ 3. Check API usage (if you use API keys)

This is separate again — only applies if you run OpenAI via API.

Go here:

👉 https://platform.openai.com/account/usage

You’ll see:

  • Token usage
  • Daily/monthly cost
  • Model breakdown
  • Billing activity

The dashboard shows usage by activity or cost across billing cycles. citeturn0search4

They are three independent meters.

⚠️ Why limits sometimes feel random

This is normal because:

  • Usage depends on context size, not message count.
  • Long coding sessions consume more allowance.
  • Limits reset on rolling windows (≈5-hour + weekly periods for Codex). citeturn0search3
