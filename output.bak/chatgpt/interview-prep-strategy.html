Harrington Interview Prep — Technical Cheat Sheet

1. ASP.NET Core Request Pipeline

Concept

ASP.NET Core uses a middleware pipeline where each component can process requests and responses.

Flow

Request    ↓ Middleware (Logging)    ↓ Middleware (Authentication)    ↓ Middleware (Authorization)    ↓ Routing    ↓ Controller / Minimal API    ↓ Response

Key idea

Middleware components run sequentially and can short-circuit the request pipeline.

Example:

app.UseAuthentication(); app.UseAuthorization(); app.MapControllers();

2. Dependency Injection (DI)

ASP.NET Core has built-in DI.

Service lifetimes:

LifetimeDescription
SingletonOne instance for entire app
ScopedOne per request
TransientNew instance every time

Example:

builder.Services.AddScoped<IOrderService, OrderService>();

Usage:

public class OrderController {     private readonly IOrderService _service;


public OrderController(IOrderService service)     {         _service = service;     } }

3. REST API Design

Principles:

  • Resource-based URLs
  • HTTP verbs
  • Stateless
  • Standard response codes

Example:

GET    /orders GET    /orders/{id} POST   /orders PUT    /orders/{id} DELETE /orders/{id}

Use DTOs to avoid exposing internal models.

4. Entity Framework Core Basics

EF Core is an ORM that maps C# objects to database tables.

Example:

public class Order {     public int Id { get; set; }     public string Customer { get; set; } }

DbContext:

public class AppDbContext : DbContext {     public DbSet<Order> Orders { get; set; } }

Query:

var orders = await _context.Orders.ToListAsync();

5. API Security

Typical API security stack:

  • HTTPS
  • JWT authentication
  • OAuth2 / OpenID Connect
  • Input validation
  • Rate limiting
  • CORS policy

Example JWT config:

services.AddAuthentication(JwtBearerDefaults.AuthenticationScheme) .AddJwtBearer(options => {     options.Authority = "https://authserver"; });

6. SQL Performance Basics

Things they may ask:

Indexes

Example:

CREATE INDEX idxorderscustomer ON Orders(CustomerId)

Avoid:

SELECT *

Prefer:

SELECT OrderId, CustomerId FROM Orders WHERE CustomerId = 5
