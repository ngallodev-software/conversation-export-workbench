On Debian 12, Docker container logs can be found in several locations depending on what you're looking for:

Primary Method - Docker Commands

The easiest way to view container logs is using Docker commands:

# View logs for a specific container docker logs <containernameor_id>


Follow logs in real-time


docker logs -f <containernameor_id>


View last 100 lines


docker logs --tail 100 <containernameor_id>


View logs with timestamps


docker logs -t <containernameor_id>

File System Locations

If you need to access the actual log files:

1. Default Docker logging driver (json-file): 

/var/lib/docker/containers/<containerid>/<containerid>-json.log

You can find these by: 

# List all container IDs docker ps -a --no-trunc


Then navigate to the directory


cd /var/lib/docker/containers/<fullcontainerid>/

2. System logs: 

/var/log/syslog  # General system logs including Docker daemon journalctl -u docker.service  # Docker daemon logs via systemd

3. Application-specific logs: Some containers mount volumes to store logs in custom locations - check your container's volume mounts: 

docker inspect <container_name> | grep -A 10 Mounts

The json-file location is most common, but keep in mind those files require root access to read directly.
