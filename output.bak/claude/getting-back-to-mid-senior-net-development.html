Great timing on this - building a solid portfolio piece will give you real talking points for interviews and demonstrate your AI-Native positioning with actual code. Let me lay out a practical plan:

Backend Foundation (.NET 9 recommended)

Core Stack:

  • .NET 9 (latest) or .NET 8 LTS if you want long-term support stability
  • ASP.NET Core Web API - use Minimal APIs for cleaner code, or Controller-based if you want something more familiar/enterprise-standard
  • Entity Framework Core with Pomelo.EntityFrameworkCore.MySql for MySQL
  • MassTransit over RabbitMQ for message queuing - it's an abstraction layer that's widely used in enterprise and makes patterns like sagas, retries, and outbox easier

Project Idea - "AI-Powered Support Ticket System":

  • Users submit support tickets via API
  • Messages get queued for processing
  • Background workers pull from queue to:

- Categorize tickets using AI (sentiment analysis, urgency scoring, routing) - Generate suggested responses - Extract key entities (product names, error codes, etc.)

  • Store everything in MySQL with proper async patterns

This hits multiple job market requirements: message queuing, background processing, AI integration, and real-world applicability.
