I'll help you split a large text file in Debian while preserving the header row. Here's a Bash script that will do this efficiently:

#!/bin/bash

Check if filename is provided

if [ $# -eq 0 ]; then echo "Usage: $0 [linesperfile]" echo "Default is 100000 lines per file if not specified" exit 1 fi

Input file

INPUT_FILE="$1"

Number of lines per file (default 100,000)

LINESPERFILE=${2:-100000}

Check if input file exists

if [ ! -f "$INPUT_FILE" ]; then echo "Error: File '$INPUT_FILE' not found." exit 1 fi

Get the header (first line)

HEADER=$(head -n 1 "$INPUT_FILE")

Split the file

- Skip the first line with tail

- Split into chunks of specified line count

- Prepend header to each output file

tail -n +2 "$INPUTFILE" | split -l "$LINESPERFILE" - "${INPUTFILE}part"

Add header back to each split file

for file in "${INPUTFILE}part_"*; do # Create a temporary file temp_file=$(mktemp)

# Write header to temp file echo "$HEADER" > "$temp_file"

# Append original split file contents cat "$file" >> "$temp_file"

# Replace original file with temp file mv "$temp_file" "$file" done

echo "File split completed. Output files are ${INPUTFILE}part_*"

Here's how to use this script:

  1. Save the script to a file, for example split_file.sh
  2. Make it executable: chmod +x split_file.sh
  3. Run it with your filename:

- Basic usage: ./split_file.sh yourfile.csv - Specify lines per file: ./split_file.sh yourfile.csv 50000

The script does these key things:

  • Extracts the first line (header) of the file
  • Splits the file into chunks (default 100,000 lines per file)
  • Adds the header back to each split file
  • Names output files with a <em>part</em> suffix

A few tips:

  • Adjust the lines per file as needed for your specific use case
  • The script works with both CSV and plain text files
  • It preserves the original file order in the split files
