When building AI agents that operate in CLI environments, minimizing output token usage is crucial for cost efficiency, latency, and staying within context limits. Here’s a structured brainstorm of strategies, with a focus on leveraging bash scripting and other practical methods.

1. Offload Formatting and Presentation to Client-Side Scripts

Instead of having the LLM generate full human-readable paragraphs or tables, have it output minimal structured data (JSON, CSV, key-value pairs). Then a lightweight bash or Python script can transform that data into a polished CLI output.

Example:

  • LLM output (low tokens):

{"status":"ok","files":["a.txt","b.txt"],"size":2048}

  • Bash wrapper:

echo "$LLM_OUTPUT" | jq -r '.files[]' | xargs ls -lh or a simple script that prints a formatted table.

Why it helps: The LLM avoids verbose narration, headers, and repetitive phrasing. The script handles the “beautification” locally, using zero model tokens.

2. Use Function Calling / Tool Use

Most modern LLM APIs support tool use (e.g., OpenAI functions, Anthropic tools). Instead of having the model describe what it would do or simulate command output, let it request that the system execute a command and return the result.

Workflow:

  • Agent decides: "I need to list files in /tmp" → calls list_files tool with optional arguments.
  • System runs ls -la /tmp and captures stdout/stderr.
  • System sends the actual output back to the model (as a tool response).
  • Model can then reason about that real output, but you can also bypass sending the full output back to the model if the agent only needs a summary.

Token savings: The model never generates the command output itself (which could be huge). It only generates the tool call and perhaps a brief follow-up.

3. Use Abbreviations, Aliases, and Shortcodes

Design the agent to output compact representations that a shell wrapper expands.

  • Example: Instead of The disk usage is 45% on /dev/sda1, output disk: /dev/sda1 45%.
  • Alias expansion: Define a set of short tokens that a script replaces with full text (e.g., {ok} → ✅ Operation completed successfully.).

Idea: Maintain a dictionary of common phrases mapped to short codes. The LLM uses the codes; the client expands them.

4. Chain Commands and Let the Shell Do the Work

Encourage the agent to output compound shell commands that themselves produce concise output. For example, instead of listing files and then summarizing counts, output:

find . -type f -name "*.log" | wc -l

The user sees only the number. The model’s output is just the command.

Token efficiency: A short command uses far fewer tokens than a natural-language description of what it does and its result.

5. Use a “Compression” Layer

If the agent must generate text, consider having it emit a compressed or abbreviated form that a script decompresses. This could be:

  • Base64 of gzipped text (though that may be inefficient for small outputs).
  • Custom shorthand (like s/pattern/replacement/ instructions for the client to apply).
  • Diff-like output where only changes from a previous state are sent.

This is more exotic but could be useful in very high-volume scenarios.

6. Stateful Sessions / Context Management

Keep conversation state on the client side. Avoid repeating the same context (e.g., system prompts, long file listings) in every exchange. Use a session ID to reference previous outputs.

  • The model can refer to “the list from earlier” instead of re-listing.
  • The client can inject relevant context only when needed.

Token saving: Less redundant input means less need for the model to re-summarize or re-state.

7. Fine-Tune a Smaller Model

Train a compact model (e.g., a 7B parameter model) specifically for CLI tasks. Because it’s specialized, it can use more concise language patterns and require fewer tokens to convey the same information.

Trade-off: Requires effort to create and maintain a fine-tuned model.

8. Use Streaming and Early Termination

If the model’s output is predictable (e.g., a command that the user will run anyway), you can stop generation after the command is emitted. The client then executes it and shows the result, bypassing any further commentary.

Implementation: Use stop sequences like \n\n or a special token to cut off generation once the actionable part is done.

9. Leverage Existing CLI Tools for Output

Instead of having the model explain how to do something, have it output the exact command to run. The user sees the command output directly from the tool, which is usually terse and well-formatted.

Example:

  • Bad (high token): “To see disk usage, you can run df -h. Here’s what it does…”
  • Good (low token): df -h

The model’s output is minimal; the tool provides the actual data.

10. Implement a Result Cache

If the same query is likely to be repeated (e.g., checking disk space every few minutes), cache the model’s response or the system’s output. Use a hash of the query to retrieve previous results without invoking the model again.

11. Use a Protocol Like JSON-RPC or gRPC

For agent-to-system communication, define a compact binary or text protocol where the model emits structured requests and the system responds with structured data. This avoids natural language entirely for routine operations.

12. Hybrid Approach: Model as “Orchestrator”

Let the model output a script (bash, Python) that the user can optionally review and run. The script contains all the logic, and the model only needs to produce it once. The user then executes the script many times, getting verbose output locally without further model cost.

Summary: Best Bets for Immediate Gains

  • Function calling / tool use – eliminates simulated output.
  • Structured output + client-side formatting – separates data from presentation.
  • Command-only responses – let the CLI tools speak for themselves.
  • Aliases and shortcodes – compress common phrases.

Combining these can dramatically reduce token usage while keeping the user experience rich. Bash scripting is perfect for the glue – parsing JSON with jq, formatting tables with column, or expanding shortcodes with sed/awk.
